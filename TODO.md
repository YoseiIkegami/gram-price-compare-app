# TODO - レビューフィードバック対応

## 優先度: 高

### 1. スマホでの入力時のズームを無効化 ✅ 対応済み
**問題**: 入力をタップするとズームになるのがうざい（スマホ）

**対応方法**:
- ✅ `viewport` の `maximum-scale=1.0, user-scalable=no` を設定
- ✅ 入力フィールドのフォントサイズを16px以上（`text-base` = 16px）に変更

**実装内容**:
- `index.html` の viewport メタタグを更新
- 入力フィールドのフォントサイズを `text-sm` から `text-base md:text-lg` に変更

**参考**: 
- iOS Safariはフォントサイズが16px未満の入力フィールドをタップすると自動ズームする
- フォントサイズを16px以上にすることで自動ズームを防げる

---

### 2. 数字を大きくして目立たせる ✅ 対応済み
**問題**: 数字もっと大きくていい（PC,スマホ余白が大きい。）もう少し目立たせる

**対応方法**:
- ✅ グラム単価の表示を大きくする（`text-2xl` から `text-4xl md:text-5xl` に変更）
- ✅ 入力フィールドのフォントサイズも大きくする（`text-sm` から `text-base md:text-lg` に変更）
- ✅ カードのパディングを調整（`p-4` から `p-6` に変更）
- ✅ PCとスマホで適切なサイズになるようレスポンシブ対応（`md:` プレフィックスを使用）

**実装内容**:
- ProductCardのグラム単価表示を `text-4xl md:text-5xl` に変更
- 入力フィールドのフォントサイズを `text-base md:text-lg` に変更
- カードのパディングを `p-6` に変更
- 入力フィールドの最小高さを `min-h-[56px]` に設定（タップ領域の確保）
- 最安商品サマリーも大きく表示（`text-3xl md:text-5xl`）

---

### 3. スクロールなしで最安商品が一目でわかる
**問題**: スクロールしなくてもどっちが安いか一目でわかるようにしたい

**対応方法**:
- 画面の上部または固定位置に最安商品のサマリーを表示
- 各商品カードのグラム単価をより大きく、目立つように表示
- 最安商品を常に画面の見える位置に配置
- 最安商品のカードを画面の上部に自動的に配置する機能

---

## 優先度: 中

### 4. スライダーで簡単に入力できるようにする
**問題**: 数字入力が面倒（スライドバーを追加して簡単に入力できるようにしてみる）

**対応方法**:
- 入力フィールドの下にスライダーを追加
- スライダーで大まかな値を設定し、入力フィールドで微調整
- スライダーの範囲を適切に設定（価格: 0-10000円、内容量: 0-10000gなど）
- スライダーと入力フィールドを連動させる

**UI案**:
- 入力フィールドの下にスライダーを配置
- スライダーを動かすと入力フィールドの値が更新される
- 入力フィールドを直接編集することも可能

## 優先度: 低

### 5. 履歴機能の追加
**問題**: 履歴機能追加

**対応方法**:
- ローカルストレージ（localStorage）に履歴を保存
- 履歴一覧画面を追加（Aboutタブの横に「履歴」タブを追加）
- 履歴から選択して再計算できるようにする
- 履歴の削除機能も追加

**データ構造案**:
```typescript
interface HistoryItem {
  id: string;
  timestamp: number;
  products: Product[];
  cheapestIndex: number | null;
}
```

**UI案**:
- タブバーに「履歴」タブを追加
- 履歴一覧を表示（日時、商品数、最安商品名）
- 履歴をタップすると計算画面に戻り、その履歴の商品が入力される
- スワイプで削除（または削除ボタン）

## 技術的な注意点

### スマホでのズーム無効化
- `viewport` の設定はアクセシビリティに配慮する
- フォントサイズを16px以上にする方法が推奨される
- `user-scalable=no` は最後の手段として使用

### レスポンシブデザイン
- PCとスマホで適切なサイズになるよう調整
- Tailwind CSSのレスポンシブクラスを活用（`md:`, `lg:`など）

### パフォーマンス
- スライダーの連動処理は `useCallback` で最適化
- 履歴の保存は `localStorage` を使用（容量制限に注意）
